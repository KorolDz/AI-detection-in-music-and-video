# AI Detection in Music and Video

Система для обнаружения фейков и цифровых манипуляций в аудио- и видеофайлах. Проект предназначен для оценки достоверности медиаконтента, выявления следов подделки и формирования отчета с результатами анализа.

## Назначение проекта

Цель проекта — разработать прикладную систему, которая:

- определяет степень достоверности аудио- и видеоконтента;
- выявляет фейки, подделки и цифровые манипуляции;
- повышает уровень информационной безопасности и доверия к цифровым источникам данных;
- поддерживает локальную обработку файлов без передачи сторонним сервисам.

Система ориентирована на использование в полуавтоматическом или автоматическом режиме и может быть интегрирована в медиа-платформы.

## Задачи системы

Проект должен обеспечивать:

- классификацию входных данных как оригинальных или модифицированных;
- анализ аудио на наличие следов генерации или редактирования голоса, шумов и тембра;
- анализ видео на наличие подмены лиц, визуальных артефактов и иных признаков цифровой манипуляции;
- использование готовых библиотек и моделей при условии корректной интеграции.

## Основные функции

- загрузка и предварительная обработка медиафайлов;
- анализ и выделение признаков подделки;
- классификация результата с указанием вероятности фальсификации;
- визуализация отчета о достоверности файла;
- хранение истории проверок и результатов анализа в базе данных `SQLite`.

## Требования к реализации

- поддержка форматов аудио: `wav`, `mp3`;
- поддержка форматов видео: `mp4`, `avi`, `mov`;
- точность классификации не ниже `85%` на тестовой выборке;
- модульная архитектура с разделением на этапы загрузки, анализа, классификации и визуализации;
- наличие пользовательского интерфейса для загрузки файлов и просмотра результатов;
- использование открытых датасетов, например `ASVspoof`, `DFDC`, `Celeb-DF`;
- выполнение вычислений локально или на серверной части без отправки файлов внешним сервисам.

## Архитектура

Проект развивается как модульная система со следующими основными частями:

- `desktop_app` — desktop-интерфейс для выбора файла, запуска анализа, просмотра результата и экспорта отчета;
- `video_detection` — модуль анализа видеофайлов и интеграции с моделью;
- `weights` — директория с весами моделей;
- `tests` — тесты для сервисного слоя, экспорта и desktop-интерфейса.

В дальнейшем архитектура может быть расширена отдельными модулями для:

- анализа аудио;
- безопасной загрузки файлов;
- хранения результатов и истории проверок в `SQLite`;
- журналирования действий;
- формирования расширенных отчетов.

## Текущий состав репозитория

На текущем этапе в проекте присутствуют:

- desktop-приложение для локального анализа видео;
- сервисный слой, преобразующий результат анализа в единый формат;
- экспорт результата в текстовый отчет;
- тесты базовой логики приложения.

## Установка и запуск

Рекомендуемая версия Python: `3.11`. Для текущего набора библиотек это самый предсказуемый вариант установки.

Установка всех зависимостей одной командой в PowerShell:

```powershell
.\scripts\setup_env.ps1
```

Скрипт:

- создаёт виртуальное окружение `.venv`;
- обновляет `pip`, `setuptools`, `wheel`;
- устанавливает все зависимости из `requirements.txt`.

Для совместимости `face_recognition_models` проект использует `setuptools` ниже версии `82`.

После установки запуск приложения:

```powershell
.\.venv\Scripts\python -m desktop_app
```

Если нужно установить зависимости вручную:

```powershell
python -m pip install -r requirements.txt
```

Основные зависимости проекта:

- `PySide6`
- `opencv-python`
- `numpy`
- `tensorflow`
- `face-recognition`
- `face_recognition_models`

Файл весов модели должен находиться в каталоге `weights/`.
